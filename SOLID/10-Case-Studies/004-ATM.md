# Case Study: ATM System

## Requirements

- Supports multiple transaction types (withdrawal, deposit, balance inquiry, transfer).
- Supports multiple card/authentication methods (PIN, biometric, chip).
- Dispenses cash using an algorithm that must work with the fewest bills possible and
  must be swappable per region (different bill denominations).

## Design

```python
from abc import ABC, abstractmethod

class AuthenticationMethod(ABC):
    @abstractmethod
    def authenticate(self, credentials) -> bool: ...

class PinAuthentication(AuthenticationMethod):
    def authenticate(self, credentials):
        return credentials.get("pin") == "1234"  # simplified

class BiometricAuthentication(AuthenticationMethod):
    def authenticate(self, credentials):
        return credentials.get("fingerprint_match", False)


class Transaction(ABC):
    @abstractmethod
    def execute(self, account, amount=None) -> str: ...

class Withdrawal(Transaction):
    def execute(self, account, amount=None):
        account.balance -= amount
        return f"Withdrew ${amount}. New balance: ${account.balance}"

class Deposit(Transaction):
    def execute(self, account, amount=None):
        account.balance += amount
        return f"Deposited ${amount}. New balance: ${account.balance}"

class BalanceInquiry(Transaction):
    def execute(self, account, amount=None):
        return f"Balance: ${account.balance}"


class CashDispenser(ABC):
    @abstractmethod
    def dispense(self, amount: int) -> dict:
        """Returns a mapping of denomination -> count."""
        ...

class USBillDispenser(CashDispenser):
    DENOMINATIONS = [100, 50, 20, 10, 5, 1]
    def dispense(self, amount):
        result = {}
        for denom in self.DENOMINATIONS:
            count, amount = divmod(amount, denom)
            if count:
                result[denom] = count
        return result


class Account:
    def __init__(self, balance):
        self.balance = balance


class ATM:
    def __init__(self, auth: AuthenticationMethod, dispenser: CashDispenser):
        self.auth = auth
        self.dispenser = dispenser

    def perform(self, credentials, account: Account, transaction: Transaction, amount=None):
        if not self.auth.authenticate(credentials):
            return "Authentication failed."
        result = transaction.execute(account, amount)
        if isinstance(transaction, Withdrawal):
            bills = self.dispenser.dispense(amount)
            print(f"Dispensing: {bills}")
        return result
```

## SOLID principles applied

- **ISP-adjacent:** `Transaction` only requires `execute`, kept minimal so any new
  transaction type (transfer, bill pay) implements exactly one method.
- **OCP:** new transaction types, new auth methods, or new regional bill dispensers
  are all added as new classes.
- **DIP:** `ATM` depends only on `AuthenticationMethod`, `CashDispenser`, and
  `Transaction` abstractions.

## A design smell in this draft — for you to fix

Notice `ATM.perform` contains `isinstance(transaction, Withdrawal)` — this is itself a
minor OCP violation (a new transaction type that also dispenses cash, like a
"partial withdrawal," would require editing this check). As an exercise, redesign
`Transaction` so dispensing behavior is part of the transaction's own contract instead
of being special-cased by `ATM`.
